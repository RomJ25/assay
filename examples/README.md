# Assay Portfolio Doctor — Manual Sample Offer

This directory contains a single end-to-end **sample report** for the Assay Portfolio Doctor service.
It exists to validate one assumption before any infrastructure gets built:

> **Will anyone pay for this artifact?**

Not "can we code it?" — clearly we can. The question is whether a public-data review report,
delivered manually, is worth real money to real investors.

## What's in this folder

| File | What it is |
|---|---|
| `sample_portfolio.csv` | A 10-ticker mixed-case demo portfolio (holdings + watchlist). |
| `portfolio_doctor_sample.md` | The artifact, in Markdown. Send-as-attachment ready. |
| `portfolio_doctor_sample.html` | The same artifact in browser-readable HTML. |
| `README.md` | This file. |

The report is generated by `scripts/generate_portfolio_doctor_sample.py`. Bear cases are templated
scaffolding from observed evidence; expect a hand-edit pass before sending to test users.

## What the report does

For each holding/watchlist name:

- Classifies it into a **review state** (sell-review / monitor / no-action / data-insufficient).
- Surfaces what changed in evidence since the prior screen (gate flips, score deltas, classification moves).
- Renders an evidence scorecard (V/Q scores, Piotroski breakdown, gates, valuation, momentum, data quality).
- Generates a structured bear case from the observed evidence.
- Lists a research checklist and decision-journal prompts.

It does **not** issue buy/sell recommendations. It does **not** predict returns. It is a
**discipline tool**, not a forecasting tool — selection alpha on the underlying screener is unproven
(see `docs/DESIGN_DECISIONS.md` "Statistical-rigor framework"). The report makes evidence visible and
easier to act on; it doesn't claim to know what the price will do next.

## The manual paid offer

To validate willingness-to-pay, the offer is delivered manually (no Stripe, no auth, no SaaS):

> Send a portfolio or watchlist CSV (ticker / position type / thesis / review date).
> Receive back a Portfolio Doctor report — public-data evidence, sell-review flags, bear cases,
> data-quality warnings — within 3 business days.

Suggested pricing for the validation phase:

| Tier | Tickers | Price |
|---|---:|---:|
| Sample | 5 | $49 |
| Standard | 15 | $99 |
| Full | 30–50 | $249 |

These prices are **deliberately set high enough to filter for real intent** and low enough that a
serious investor reviewing a real portfolio would consider them. The point is signal, not revenue scale.

## Generating a fresh report

```bash
# 1. Run a fresh screen (populates RESEARCH CANDIDATE labels, data_quality, gate audit trail).
python main.py --top 500

# 2. Generate the report from the latest two full screens.
python scripts/generate_portfolio_doctor_sample.py

# Or specify portfolios/snapshots explicitly:
python scripts/generate_portfolio_doctor_sample.py \
  --portfolio examples/my_portfolio.csv \
  --current results/screen_2026-04-26.json \
  --prior results/screen_2026-04-05.json
```

## What this is **not**

- Not financial advice. Not a buy/sell recommendation.
- Not a stock picker. The screener's selection alpha is unproven and should not be sold as such.
- Not a SaaS yet. Manual delivery is intentional — it forces the artifact to stand on its own merits.
- Not a backtest. The report describes the *current* evidence state and what changed since the prior
  snapshot; it does not claim historical performance.

If users pay for the artifact, build Sprint 1. If users don't pay, the strategy needs another pivot.
